## 2024-04-04 - Semantic Interactive Elements for Radix Triggers
**Learning:** Radix UI components (like `PopoverTrigger` with `asChild`) expect semantically correct and focusable interactive child elements to work smoothly with accessibility out of the box. Using non-interactive `<span>` tags as triggers limits screen reader compatibility and keyboard navigation.
**Action:** When a static element (like an icon `<span>`) becomes interactive (e.g. by passing an `onChange` handler to a component), it should be swapped to a semantically correct interactive element like a `<button type="button">`. Always pair interactive elements without text content with a descriptive `aria-label`.
